## Changes

### SW-BE-002 — Pagination and stable sorting (`backend/src/modules/auth/dto/`)

- **`get-refresh-tokens.dto.ts`** — New `GetRefreshTokensDto` that extends the
  shared `PaginationDto` with a typed `RefreshTokenSortField` enum allowlist
  (`createdAt | lastUsedAt | expiresAt`). Arbitrary column names are rejected
  at validation time, preventing schema-probing via query parameters.
- **`auth-pagination-sort.spec.ts`** — 15 unit tests covering: every valid
  sort field, SQL-injection rejection, default values, pagination constraints
  (page ≥ 1, limit 1–100), sort order (ASC/DESC), and enum stability.

### SW-BE-003 — Idempotency and replay tests (`backend/src/modules/auth/`)

- **`auth-idempotency-replay.spec.ts`** (pre-existing) — Verified complete.
  Covers login idempotency (independent token pairs per call), refresh
  single-use enforcement, concurrent refresh race (family revocation), logout
  idempotency, JWT replay/tamper scenarios, and no-secrets-in-logs guarantee.
- **`backend/jest.config.ts`** — Added `prom-client` to `moduleNameMapper` so
  the spec suite resolves correctly in CI without needing `prom-client`
  installed as a test dependency.

### SW-BE-004 — DTO validation and error mapping (`backend/src/modules/auth/dto/`)

- **`auth-dto-validation.spec.ts`** (pre-existing) — Verified complete.
  Covers `LoginDto`, `AdminLoginDto`, `RefreshTokenDto`, `WalletLoginDto` (all
  constraints, transforms, and edge cases) plus `AllExceptionsFilter`
  validation-array-to-string mapping.

### SW-BE-005 — Audit trail hooks (`backend/src/modules/auth/audit/`)

- **`auth-audit.events.ts`** (pre-existing) — Canonical event constants.
- **`auth-audit.service.ts`** (pre-existing) — `AuthAuditService` with email
  redaction and WARN/LOG routing for security vs. informational events.
- **`auth-audit.spec.ts`** (pre-existing) — Verified complete. 44 tests
  covering event routing, email redaction, no-raw-token guarantees, and
  AuthService integration (every auth flow fires the correct audit event).

## Rollout / migration notes

- **No schema changes** — `GetRefreshTokensDto` is a query DTO only; it does
  not alter the `refresh_tokens` table.
- **No feature flags needed** — pagination/sorting is additive; existing
  consumers of the auth endpoints are unaffected.
- **Backward-compatible** — all new validators have `@IsOptional()` with
  sensible defaults.
- **Env vars unchanged** — no new environment variables.

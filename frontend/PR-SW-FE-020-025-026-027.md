## Changes

### SW-FE-020 — Shop Grid CLS / LCP budget

**`frontend/src/hooks/useShopGridWebVitals.ts`** (new)
- Monitors `largest-contentful-paint` and `layout-shift` via `PerformanceObserver`.
- Budget: LCP < 2 500 ms, CLS < 0.1 — matching the existing `useJoinRoomWebVitals` pattern.
- Reports to `/api/v1/metrics` in production; debug-logs in dev; silently no-ops in jsdom.
- Cleans up observers and `beforeunload` listener on unmount.

**`frontend/src/components/game/ShopGrid.tsx`** (updated)
- Wires `useShopGridWebVitals()` into the component (one line, no new deps).
- Skeleton cards already carry `min-h-[160px]` to pin layout dimensions and prevent CLS.

**`frontend/src/components/game/ShopGrid.test.tsx`** (updated)
- Adds `vi.mock("@/hooks/useShopGridWebVitals", ...)` to keep the existing test suite clean.

**`frontend/test/ShopGrid.test.tsx`** (updated)
- Same mock addition.

### SW-FE-025 — Purchase Modal accessibility and focus order

No new code required — `PurchaseModal` already satisfies every WCAG 2.1 requirement:

- `role="dialog"`, `aria-modal="true"`, `aria-labelledby`, `aria-describedby`.
- `useFocusTrap` — moves focus to first focusable element on open, wraps Tab/Shift+Tab, closes on Escape, restores prior focus on close.
- Body scroll lock with prior-value restore.
- All icons marked `aria-hidden="true"`.
- Backdrop marked `aria-hidden="true"`.
- Price region has `aria-live="polite"` and `aria-atomic="true"`.

Verification: covered by new tests in `frontend/test/PurchaseModal.test.tsx`.

---

### SW-FE-026 — Purchase Modal TypeScript strictness and null guards

No new production code required — already hardened:

- `PurchaseModalProps` uses `readonly` on every prop.
- Nullable props typed as `string | null | undefined`.
- `toSafeString(value)` coerces null/undefined → `''` and strips HTML tags.
- `safeName`, `safePrice`, `safeCurrency` used everywhere instead of raw props.
- All hooks called before the conditional `if (!isOpen) return null`.

Verification: new tests cover `itemName: null`, `itemName: ''`, `itemName: undefined`, `itemPrice: null`, `itemCurrency: null`, and XSS (HTML-stripping) for both name and price.
